# Software Design

## target audience - juniors
This course is for undergrad cs students 3rd year

## prerequisites

- Basic understanding of programming concepts
- Familiarity with at least one programming language (e.g., Python, Java, C++)
- Knowledge of data structures and algorithms
- Exposure to software development life cycle (SDLC) concepts
- Eagerness to learn and experiment with design patterns

## learning objectives

- Grasp core principles of software design and architecture.
- Apply design patterns effectively in software development.
- Analyze and evaluate software solutions for scalability and maintainability.
- Implement UML diagrams to visualize software architecture.
- Collaborate using Agile methodologies in team projects.
- Conduct code reviews to ensure adherence to design principles.
- Deploy basic testing strategies to validate software designs.

## course outline
#### chapter 1 - introduction to software design
- objectives: Understand the importance of software design and architecture in developing high-quality software.
- outline:
  - Definition of software design
  - Importance of good design
  - Overview of software architecture vs. software design

#### chapter 2 - software design principles
- objectives: Explore key principles that guide effective software design.
- outline:
  - SOLID principles
  - DRY (Don't Repeat Yourself) principle
  - KISS (Keep It Simple, Stupid) principle
  - YAGNI (You Aren't Gonna Need It) principle

#### chapter 3 - design patterns in software development
- objectives: Learn about various design patterns and their applications.
- outline:
  - Introduction to design patterns
  - Creational patterns (e.g., Singleton, Factory)
  - Structural patterns (e.g., Adapter, Composite)
  - Behavioral patterns (e.g., Observer, Strategy)

#### chapter 4 - software architecture
- objectives: Understand the different architectures and their relevance in software design.
- outline:
  - Overview of software architecture
  - Monolithic architecture
  - Microservices architecture
  - Event-driven architecture and its impact on scalability

#### chapter 5 - UML and visualization techniques
- objectives: Learn to create and interpret UML diagrams for effective communication of software designs.
- outline:
  - Introduction to UML (Unified Modeling Language)
  - Class diagrams
  - Sequence diagrams
  - Use case diagrams

#### chapter 6 - Agile methodologies and collaboration
- objectives: Explore Agile practices and tools that enhance team collaboration in software projects.
- outline:
  - Overview of Agile methodologies (Scrum, Kanban)
  - Roles in Agile teams
  - Tools for collaboration (JIRA, Trello)

#### chapter 7 - evaluating software solutions
- objectives: Develop skills to analyze software designs for scalability and maintainability.
- outline:
  - Key metrics for evaluating design
  - Case studies of good vs. poor software design
  - Techniques for assessing maintainability

#### chapter 8 - conducting code reviews
- objectives: Learn the importance of code reviews and how to conduct them effectively.
- outline:
  - Purpose of code reviews
  - Best practices in reviewing code
  - Tools and frameworks for code review

#### chapter 9 - testing strategies for software designs
- objectives: Introduce basic testing strategies that complement software design for quality assurance.
- outline:
  - Types of testing (unit, integration, system)
  - Test-driven development (TDD)
  - Writing effective test cases

#### chapter 10 - real-world software design case studies
- objectives: Analyze real-world examples to reinforce design principles and practices.
- outline:
  - Case studies of successful software projects
  - Lessons learned from design failures
  - Group discussion on design decisions made in case studies

#### chapter 11 - prototyping and iterative design
- objectives: Understand the role of prototyping and iterations in refining software designs.
- outline:
  - Importance of prototyping
  - Techniques for creating prototypes (low-fidelity vs. high-fidelity)
  - Iterative design process and feedback loops

#### chapter 12 - capstone project
- objectives: Apply knowledge and skills from the course in a real-world software design project.
- outline:
  - Project overview and requirements
  - Group formation and roles
  - Presentation of designs and peer feedback
  - Reflection on the design process and lessons learned
